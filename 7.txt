1. Database Syncing

Aurora Global Database

Best for multi-region apps needing low-latency reads and fast disaster recovery.

1 primary region, up to 5 read-only replicas in other regions.

Traditional RDS

Can create read replicas, but slower replication (seconds to minutes).

Not ideal for real-time sync across continents.

2. Object Storage

S3 Cross-Region Replication (CRR)

Automatically replicates objects to another region.

Useful for global apps (download files/images/videos from nearest location).

3. Caching Layer

CloudFront: Global CDN with edge caches → reduces latency for static + dynamic content.

MemoryDB / ElastiCache (Redis/Memcached): In-memory DB for microsecond response time.

Helps reduce database load.

Multi-region deployment possible (but needs careful sync setup).

4. Configs & Secrets

Secrets Manager replication for DB/API keys.

Parameter Store global configs for uniform settings across clusters.

5. Other sync needs

EKS Configs (ConfigMaps/Secrets) → replicate across regions manually or using CI/CD.

Logging/Monitoring → use CloudWatch + OpenTelemetry to unify logs globally.
